## BUG 1: Notifikační dialog — tlačítka "Zahodit" / "Zkusit znovu" nereagují ✅ OPRAVENO

**Priorita**: HIGH

### Problém

Kliknutí na "Zahodit" nebo "Zkusit znovu" v error notifikaci vizuálně nic neudělá.
Dialog zůstane otevřený, žádná zpětná vazba uživateli.

### Oprava

`_userTaskDialogEvent.value = null` přesunuto z `try` do `finally` bloku
v obou metodách `discardTask()` a `retryTask()`. Dialog se teď zavře
i při selhání backend volání.

**Soubor**: `shared/ui-common/.../notification/NotificationViewModel.kt`


## BUG 2: Spring Data "Parameter does not have a name" ✅ OPRAVENO

**Priorita**: CRITICAL

### Problém

Kotlin inline value class mangluje parameter names v bytecode,
Spring Data nenajde parameter name → CRASH při čtení TaskDocument.

### Oprava

Přidán `@PersistenceCreator` companion factory `create()` s primitivními typy
(ObjectId, String) pro všechny constructor parametry. Spring Data nyní používá
tuto factory místo přímého constructoru s value class parametry.

**Soubor**: `backend/server/.../entity/TaskDocument.kt`

**Poznámka**: `FilteringRuleDocument.kt` tento problém nemá — používá raw ObjectId/String.


## BUG 3: K8s namespace "bms-commerzbank" not found ✅ OPRAVENO

**Priorita**: MEDIUM

### Problém

Při syncu prostředí se PATCH ConfigMap pokouší pracovat s namespace,
který neexistuje (smazán ručně nebo nebyl vytvořen).

### Oprava

1. Přidána metoda `ensureNamespaceExists()` — auto-vytvoří namespace pokud chybí
2. Volána na začátku `syncEnvironmentResources()` před jakýmikoli K8s operacemi
3. `createOrUpdateConfigMap()` — přidáno zachycení 404 s jasnou error zprávou

**Soubor**: `backend/server/.../service/environment/EnvironmentK8sService.kt`

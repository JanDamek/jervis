## Bug 1: UserTaskListItemDto ClassNotFoundException

### Problém
Po startu desktop app se zobrazí macOS error dialog:
```
com/jervis/dto/user/UserTaskListItemDto
```

### Příčina
Desktop app binary je z 19.2., ale `UserTaskListItemDto` bylo přidáno 23.2. (commit `2cc90b77`).
Server posílá CBOR odpověď s `UserTaskListItemDto`, klient nemá třídu → deserializace selže.

### Fix
1. **Okamžitý**: Přebuildit desktop app (`k8s/build_desktop.sh`)
2. **Preventivní**: Graceful handling verze mismatch — server by měl posílat verzi API v handshake, klient by měl ukázat "Aktualizujte aplikaci" místo raw exception

### Soubory
- `shared/common-dto/.../user/UserTaskDto.kt` — `UserTaskListItemDto` definice (řádek 23)
- `shared/common-api/.../IUserTaskService.kt` — `listAllLightweight()` RPC metoda
- Desktop app packaging / build script

## Bug 2: "Zkusit znovu" u workspace preparation nic nedělá

### Problém
Banner "Příprava prostředí selhala" s chybou:
```
Connection not found: project=nUFO resource=mazlusek/moneta/nufo connectionId=6986002d8bf32b35197e2bf2
```

Tlačítko "Zkusit znovu" nedá žádný feedback — ani success, ani error.

### Příčina

1. **Connection chybí v DB** — projekt odkazuje na neexistující connectionId
2. **Retry selže se stejnou chybou** — connection stále neexistuje
3. **Error je spolknut potichu**:
   ```kotlin
   // MainViewModel.kt:480
   fun retryWorkspace() {
       scope.launch {
           try {
               repository.projects.retryWorkspace(projectId)
           } catch (_: Exception) {
               // Ignore — will retry automatically via backoff
           }
       }
   }
   ```
4. **CLONE_FAILED_NOT_FOUND je non-retryable** — BackgroundEngine nemá auto-retry pro tento stav, ale UI to neříká

### Řešení

#### 2.1 Feedback pro uživatele
Místo tichého `catch` → zobrazit snackbar s chybou:
```kotlin
catch (e: Exception) {
    _snackbarMessage.value = "Opakování selhalo: ${e.message}"
}
```

#### 2.2 Rozlišit retryable vs non-retryable v UI
Pokud stav = `CLONE_FAILED_NOT_FOUND` nebo `CLONE_FAILED_AUTH`:
- Zobrazit "Zkontrolujte připojení v nastavení projektu" místo "Zkusit znovu"
- Nebo tlačítko "Opravit" → navigovat do nastavení připojení projektu

#### 2.3 Banner s kontextem
```
⚠️ Příprava prostředí selhala
   Připojení "mazlusek/moneta/nufo" nebylo nalezeno.
   [Opravit připojení]  [Ignorovat]
```

### Soubory
- `shared/ui-common/.../MainScreen.kt` — WorkspaceBanner (řádky 337-389)
- `shared/ui-common/.../MainViewModel.kt` — retryWorkspace(), WorkspaceInfo (řádky 109-127, 480-491)
- `backend/server/.../service/indexing/git/GitRepositoryService.kt` — ensureAgentWorkspaceReady (řádek 259)
- `backend/server/.../service/background/BackgroundEngine.kt` — initializeProjectWorkspace (řádek 1390)

## Bug 3: Datový problém — chybějící connection pro nUFO

### Problém
Projekt nUFO má git resource `mazlusek/moneta/nufo` s `connectionId=6986002d8bf32b35197e2bf2`, ale connection v DB neexistuje.

### Datový fix
Zkontrolovat v MongoDB:
```bash
mongosh --eval "db.connections.findOne({_id: ObjectId('6986002d8bf32b35197e2bf2')})"
```

Pokud neexistuje:
- Buď znovu vytvořit connection v UI (Nastavení → Připojení)
- Nebo odebrat git resource z projektu nUFO

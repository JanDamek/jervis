# Jervis monorepo — shared Python dependency version constraints.
#
# Single source of truth for versions of libraries that appear in multiple
# service requirements.txt files. Per-service requirements.txt keeps `>=`
# (flexible) entries; Docker builds pin them via `pip install -c` against
# this file so every service resolves the same version.
#
# Update here, rebuild all services — no per-service drift.
#
# Usage in service Dockerfile:
#   COPY constraints.txt /constraints.txt
#   RUN pip install --no-cache-dir -r requirements.txt -c /constraints.txt
#
# Dev workspace (uv sync) picks these up via root [tool.uv] constraint-dependencies.

# in dependencies we need; constraints just pin the base-package version.
